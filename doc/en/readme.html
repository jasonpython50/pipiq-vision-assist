PiPiQ Vision Assist

Vision assistance for blind users, powered by OpenCode Go vision AI models. Version 1.3.0. Author: Emmanuel Asamoah.

The addon gives you five abilities:

  1. QR code locator: finds a QR code on your screen (WhatsApp Web, websites, anywhere) and tells you exactly where to point your phone camera, in plain words such as "top right of the screen, 78 percent from the left edge". It can also park the mouse pointer on the code so you can feel where it is with mouse tracking.
  2. Image describer: describes any image with AI: the object under the NVDA navigator, the whole screen, an image on the clipboard, or any image you pick from a list of all images on the current web page.
  3. Screenshot taker: photographs the navigator object, the current window, or the whole screen at full resolution, saves the picture as a PNG file in your Pictures folder, and copies it to the clipboard so you can paste it straight into a chat or email.
  4. Capture check: before you photograph or describe something, it tells you whether the object is fully on screen and whether another window is covering it.
  5. Form check: when a Next or Submit button on a web form is dimmed and the page gives no hint why, press the Vision assist gesture and then F to hear exactly what the form still needs: which required fields are empty, which checkboxes are unchecked, and what the page marks as invalid.

Getting started

  1. Install the addon and restart NVDA when prompted.
  2. Open the NVDA menu, then Preferences, then Settings, then the PiPiQ Vision Assist category.
  3. Paste your OpenCode API key into the "OpenCode API key" field and press OK. You can get a key by subscribing to OpenCode Go at opencode.ai.

Taking screenshots works without an API key. The QR locator and the image describer need the key, because they send the image to the AI service.

Privacy

  • For QR detection and image descriptions, screenshots and clipboard images are sent to the OpenCode API for analysis. Do not run those commands while confidential content is on screen if that concerns you.
  • Screenshots taken with the T commands stay on your computer; nothing is uploaded.
  • The API key is stored in NVDA's configuration file in plain text, like most NVDA addon keys.

Commands

Everything starts with one gesture: NVDA+Shift+0 (the zero on the number row). You can change it in the Input Gestures dialog, category "PiPiQ Vision Assist". After pressing it, press one key:

Vision assist layer commands
KeyAction
QFind a QR code on the whole screen
WFind a QR code in the current window only
ODescribe the image at the navigator object with AI
SDescribe the whole screen with AI
CDescribe the image on the clipboard with AI (works with copied images and copied image files)
GList all images on the current web page and choose one to describe with AI
FCheck the form on the current web page: reports dimmed buttons and which required fields are still empty, unchecked, or marked invalid
PCheck what a screenshot or description would capture: the object's size, whether it is fully on screen, and whether another window covers it
TTake a screenshot of the navigator object: save it to Pictures and copy it to the clipboard
Shift+TTake a screenshot of the current window
Control+TTake a screenshot of the whole screen
RRepeat the last result
BOpen the last result in a browseable window, so you can arrow through it line by line; Escape closes it
H or F1Speak this command list
EscapeCancel a running request, or close the layer

Every command can also be bound to its own direct gesture in the Input Gestures dialog.

While a request runs you hear a soft beep every second (this can be turned off in settings). A high beep means success, a low beep means failure, and the result or error is spoken. Press the entry gesture and then the same command again to cancel and restart if it takes too long.

Describing an image on a web page

  1. Move focus to the web page in your browser.
  2. Press NVDA+Shift+0, then G.
  3. A list of every image on the page opens, using each image's alternative text as its label. Images with no alternative text are listed as "Unlabeled image", and describing exactly those is often the point.
  4. Arrow to the image you want and press Enter. The addon scrolls the image into view, photographs it, and reads you the AI description.

Tip: the picture must be visible on screen to be photographed, so this works best with the browser window maximized. If the addon reports that the image could not be brought into view, scroll to it manually and press NVDA+Shift+0 then O on it instead.

Checking why a form button is dimmed

You have filled in an online form, but the Proceed or Submit button stays dimmed and NVDA gives no hint why. The form check finds out for you:

  1. Stay on the web page, ideally with the browse mode cursor somewhere inside the form.
  2. Press NVDA+Shift+0, then F.
  3. The addon walks through every field of the form (or of the whole page, when the cursor is not inside a form) and reads a report such as: "The Proceed button is dimmed and cannot be pressed yet. After checking 9 fields in the current form, 2 likely reasons were found. 1: Phone number, edit box, is required but still empty. 2: I agree to the terms of service, checkbox, is required but not checked."

The report covers dimmed buttons, required fields that are still empty, required checkboxes and radio button groups with nothing selected, and fields the page marks as invalid, including any explanation the page attaches to the field, such as "Password must be at least 8 characters". Long reports open in a browseable window, and R and B re-read the last report like every other result.

The check can only see what the page tells screen readers. If a site never marks its fields as required, or the button is dimmed because of a CAPTCHA or a verification code you must request first, the report says that everything looks complete and suggests describing the screen with S, so the AI can look for purely visual error messages.

Taking screenshots

Press NVDA+Shift+0, then T for the navigator object, Shift+T for the current window, or Control+T for the whole screen. The addon announces the file name and the picture's size in pixels. The picture is:

  • saved as a PNG file in the PiPiQ Screenshots folder inside your Pictures folder, named with the date and time, for example pipiq-2026-07-13-14-30-05.png, and
  • copied to the clipboard, so you can paste it directly into WhatsApp, an email, or a document with Control+V.

Before shooting, you can press P to hear whether the object is fully on screen and whether another window would appear in the picture. Screenshots are refused while NVDA's screen curtain is active, because they would be completely black.

Scanning a WhatsApp Web QR code, step by step

  1. Open web.whatsapp.com and maximize the browser window (Windows+Up arrow).
  2. Press NVDA+Shift+0, then W.
  3. Wait for the high beep. You will hear something like: "QR code found in the middle right of the window, centered 75 percent from the left edge and 50 percent from the top. It spans about 20 percent of the width. The mouse pointer is now on the QR code."
  4. On your phone, open WhatsApp, then Settings, then Linked devices, then Link a device, and point the camera at the reported part of your computer screen. Hold the phone 20 to 40 centimeters away, roughly parallel to the screen. Raising the screen brightness helps.
  5. If it does not catch, press NVDA+Shift+0 then W again; the position may shift when the site refreshes the code.

Settings

All settings live under NVDA menu, Preferences, Settings, PiPiQ Vision Assist:

  • OpenCode API key: your key, starting with sk-.
  • API base URL: default https://opencode.ai/zen/go/v1. Any OpenAI-compatible server also works.
  • Model for QR code detection: default qwen3.7-plus (tested: accurate boxes, no false positives).
  • Model for image descriptions: default kimi-k2.5 (tested: fast, high quality descriptions).
  • Refresh model list from server: downloads the current model list with your key; vision-capable models are marked.
  • Show only vision-capable models: hides text-only models that cannot analyze images.
  • What to do with images: Automatic (default: pictures get described, text-heavy screenshots get their text read out exactly as written), Always describe, or Always extract the text exactly.
  • Image description detail: brief, detailed, or your own custom prompt.
  • Request timeout: default 90 seconds. QR detection on busy screens can take 20 to 60 seconds depending on the model.
  • Maximum image size: images are downscaled to this many pixels on the longest edge before uploading to the AI. Smaller is faster, larger preserves fine detail. Saved screenshots are always full resolution.
  • Move the mouse pointer to a found QR code: on by default.
  • Presenting results: speak only, speak and open a window for long results (default), or always open the window.
  • Progress beeps and response language: self explanatory.

Troubleshooting

  • "The API rejected your key": re-paste the key in settings and make sure your OpenCode Go subscription is active.
  • "No QR code was found" but you expect one: the code may be scrolled out of view or behind another window. Bring it on screen, then try Q (whole screen) instead of W.
  • "The captured image is completely black": NVDA's screen curtain or a privacy filter is active; turn it off and try again.
  • Descriptions come in English: check "Ask for descriptions in your NVDA language" in settings.
  • An image on a web page cannot be chosen: some decorative images are hidden from screen readers and do not appear in the list; there is nothing meaningful in them to describe.
  • The form check says everything looks complete but the button stays dimmed: the page is not exposing the real requirement to screen readers. Press NVDA+Shift+0 then S to have the AI describe the screen; visual error messages and CAPTCHAs show up there.
  • Detailed errors are written to the NVDA log, which you can open with NVDA+F1.
